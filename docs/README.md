# Project Documentation

This folder collects generated project documentation, transition plans, architecture notes, reports, ADRs, and checkpoints for KuoteSuite and Pulse.

The repository root keeps only conventional project-level files such as `README.md` and `LICENSE`.

## Pulse Transition

- `PULSE_TRANSITION_PLAN.md`: initial plan for moving from KuoteSuite toward Pulse.
- `adr/0001-new-pulse-monorepo.md`: decision to create a new Pulse monorepo scaffold beside the existing prototype.

## Brand

- `brand/BRAND_STANDARDS.md`: baseline Pulse visual standards, including Manrope as the default font.

## Architecture

- `architecture/DATABASE_MIGRATION_PLAN.md`: PostgreSQL and Prisma migration path for the existing KuoteSuite backend.
- `architecture/PULSE_LEADS_MODULE_DRAFT_DESIGN.md`: research-backed draft design for the Pulse Leads module.

## Checkpoints

- `checkpoints/RESTART_CHECKPOINT.md`: latest saved restart/checkpoint state.

## Reports

- `reports/PROJECT_ASSESSMENT.md`: inherited project assessment.
- `reports/AUTH_REFACTOR_REPORT.md`: authentication refactor state.
- `reports/DEPENDENCY_MODERNIZATION_REPORT.md`: dependency modernization report.
